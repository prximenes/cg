Parte 1 — Rasterização Canvas 2D

Conceito central: converter geometria matemática contínua (retas, círculos) em pixels discretos numa grade bidimensional.

A tela de um computador é uma grade de pixels. Quando pedimos para desenhar uma reta de (0,0) a (800,600), o código precisa decidir quais dos milhões de pixels devem ser acesos. Esse processo é a rasterização.

1.1 — DDA (Digital Differential Analyzer)

Arquivo: rasterizacao/dda.html

O DDA avança passo a passo ao longo do eixo dominante (aquele com maior variação), calculando a posição no eixo secundário como número de ponto flutuante e arredondando para o pixel mais próximo.

Matemática

dx = x2 - x1 (variação total em x) dy = y2 - y1 (variação total em y) steps = max(|dx|, |dy|) (nº de pixels = eixo dominante) xInc = dx / steps (incremento em x por passo — pode ser fração) yInc = dy / steps (incremento em y por passo — pode ser fração) para i de 0 até steps: pixel( round(x), round(y) ) x ← x + xInc y ← y + yInc

A divisão por steps garante que o eixo dominante avance exatamente 1 pixel por passo, enquanto o eixo secundário avança proporcionalmente.

Implementação — dda.html (linhas 238–290)

javascript rasterizacao/dda.html 
function algoritmoDDA(x1, y1, x2, y2) {
  const dx    = x2 - x1;
  const dy    = y2 - y1;
  const steps = Math.max(Math.abs(dx), Math.abs(dy));

  // Incrementos fracionários: eixo dominante avança 1/passo
  const xInc = dx / steps;
  const yInc = dy / steps;

  let x = x1;
  let y = y1;

  for (let i = 0; i <= steps; i++) {
    // Arredonda para o pixel inteiro mais próximo
    ctx.fillRect(Math.round(x), Math.round(y), 1, 1);

    x += xInc;   // avança fração em x
    y += yInc;   // avança fração em y
  }
}

1.2 — Bresenham para Retas

Arquivo: rasterizacao/bresenham_reta.html

Jack Bresenham (1962) percebeu que multiplicando a comparação por 2·dx é possível tomar a mesma decisão do DDA usando apenas inteiros: somas e subtrações, sem divisão, sem ponto flutuante.

A variável de decisão P

A cada passo no eixo x, precisamos decidir: o próximo pixel é (x+1, y) ou (x+1, y+1)? O parâmetro acumulado P responde isso:

Parâmetro inicial (1º octante): p₀ = 2·dy − dx A cada passo em x: se p ≥ 0: y++ (reta passou acima — pixel de cima) p ← p + 2·(dy − dx) se p < 0: y não muda (reta ainda abaixo — mesmo y) p ← p + 2·dy
A multiplicação por 2·dx (implícita acima) elimina a divisão. Resultado: apenas somas e subtrações de inteiros.

Generalização para 8 octantes

dx = |x2 - x1|, sx = sinal(x2 - x1) (+1 ou -1) dy = |y2 - y1|, sy = sinal(y2 - y1) (+1 ou -1) se |dx| ≥ |dy|: eixo dominante = X p₀ = 2·dy − dx loop de 0 até dx: pinta (x, y) se p ≥ 0: y += sy; p += 2·(dy − dx) senão: p += 2·dy x += sx ← eixo dominante sempre avança se |dy| > |dx|: eixo dominante = Y (troca de eixo) p₀ = 2·dx − dy loop de 0 até dy: pinta (x, y) se p ≥ 0: x += sx; p += 2·(dx − dy) senão: p += 2·dx y += sy

Implementação — bresenham_reta.html (linhas 269–344)

javascript rasterizacao/bresenham_reta.html 
function bresenhamReta(x1, y1, x2, y2) {
  let dx = Math.abs(x2 - x1);
  let dy = Math.abs(y2 - y1);
  let sx = (x2 >= x1) ? 1 : -1;  // sinal de avanço em x
  let sy = (y2 >= y1) ? 1 : -1;  // sinal de avanço em y

  let x = x1, y = y1;

  if (dx >= dy) {
    // Eixo dominante = X
    let p = 2 * dy - dx;          // parâmetro de decisão inicial

    for (let i = 0; i <= dx; i++) {
      ctx.fillRect(x, y, 1, 1);   // pinta pixel

      if (p >= 0) {
        y += sy;                  // y sobe (ou desce)
        p += 2 * (dy - dx);       // atualização com mudança de y
      } else {
        p += 2 * dy;              // atualização sem mudança de y
      }
      x += sx;                    // eixo dominante sempre avança
    }
  } else {
    // Eixo dominante = Y (papéis de x e y trocados)
    let p = 2 * dx - dy;

    for (let i = 0; i <= dy; i++) {
      ctx.fillRect(x, y, 1, 1);

      if (p >= 0) {
        x += sx;
        p += 2 * (dx - dy);
      } else {
        p += 2 * dx;
      }
      y += sy;
    }
  }
}

1.3 — Bresenham para Círculos

Arquivo: rasterizacao/bresenham_circulo.html

Simetria de 8 octantes

Um círculo possui simetria octafold: o arco de 0° a 45° (1º octante) é suficiente para reconstruir os 360°. Para cada ponto (x, y) calculado, geramos 8 pixels de graça por reflexão.

Dado o ponto (x, y) relativo ao centro (cx, cy): Octante 1: (cx+x, cy−y) Octante 5: (cx−x, cy+y) Octante 2: (cx+y, cy−x) Octante 6: (cx−y, cy+x) Octante 3: (cx+y, cy+x) Octante 7: (cx−y, cy−x) Octante 4: (cx+x, cy+y) Octante 8: (cx−x, cy−y)

Variável de decisão e derivação

Interpretação geométrica: p = x² + y² − r² Se p < 0: ponto dentro do círculo → y não muda Se p ≥ 0: ponto fora do círculo → y decrementa Parâmetro inicial (topo do círculo, x=0, y=r): p₀ = 1 − r A cada passo (enquanto x < y): x++ se p < 0: y não muda p ← p + 2·x + 1 (Δx² entre passos) se p ≥ 0: y−− p ← p + 2·(x − y) + 1 (Δx² − Δy² entre passos) pinta os 8 octantes

Implementação — bresenham_circulo.html (linhas 149–215)

javascript rasterizacao/bresenham_circulo.html 
function bresenhamCirculo(cx, cy, r, cor) {
  // Pinta os 8 pixels simétricos de um só passo
  function pintaOctantes(x, y) {
    const pts = [
      [cx + x, cy - y],  [cx + y, cy - x],
      [cx + y, cy + x],  [cx + x, cy + y],
      [cx - x, cy + y],  [cx - y, cy + x],
      [cx - y, cy - x],  [cx - x, cy - y],
    ];
    pts.forEach(([px, py]) => ctx.fillRect(px, py, 1, 1));
  }

  let x = 0;
  let y = r;
  let p = 1 - r;         // parâmetro de decisão inicial

  pintaOctantes(x, y);   // pinta ponto inicial (topo)

  while (x < y) {        // percorre até 45° (x == y)
    x++;

    if (p < 0) {
      p = p + 2 * x + 1;           // y não muda
    } else {
      y--;
      p = p + 2 * (x - y) + 1;    // y decrementou
    }

    pintaOctantes(x, y);
  }
}

Parte 2 — Curvas de Bézier Canvas 2D

Arquivo: curvas/bezier.html

Uma curva de Bézier é definida por pontos de controle que atraem a curva sem necessariamente tocá-la. A curva passa obrigatoriamente pelos pontos extremos (P₀ e Pₙ) e é contida no fecho convexo dos pontos de controle.

Fórmula Geral — Polinômios de Bernstein

Para n+1 pontos de controle P₀, P₁, …, Pₙ e parâmetro t ∈ [0,1]:

n B(t) = Σ C(n,i) · ti · (1−t)(n−i) · Pᵢ i=0 onde C(n,i) = n! / (i! · (n−i)!) (coeficiente binomial) Grau 1 — linear (2 pontos): B(t) = (1−t)·P₀ + t·P₁ Grau 2 — quadrática (3 pontos): B(t) = (1−t)²·P₀ + 2·(1−t)·t·P₁ + t²·P₂ Grau 3 — cúbica (4 pontos): B(t) = (1−t)³·P₀ + 3·(1−t)²·t·P₁ + 3·(1−t)·t²·P₂ + t³·P₃

Implementação — bezier.html

javascript curvas/bezier.html 
// Coeficiente binomial C(n,k) — iterativo, sem fatorial
function binomial(n, k) {
  if (k < 0 || k > n) return 0;
  if (k === 0 || k === n) return 1;
  let resultado = 1;
  for (let i = 0; i < k; i++) {
    resultado = resultado * (n - i) / (i + 1);
  }
  return Math.round(resultado);
}

// Ponto na curva de Bézier dado t ∈ [0,1]
function bezier(t, pts) {
  const n = pts.length - 1;   // grau = nº de pontos - 1
  let bx = 0, by = 0;

  for (let i = 0; i <= n; i++) {
    // Peso de Bernstein para o i-ésimo ponto de controle
    const w = binomial(n, i)
            * Math.pow(t, i)
            * Math.pow(1 - t, n - i);
    bx += w * pts[i].x;
    by += w * pts[i].y;
  }
  return { x: bx, y: by };
}

// Renderiza avaliando 'resolucao' pontos em t ∈ [0,1]
for (let i = 0; i <= resolucao; i++) {
  const t = i / resolucao;
  const p = bezier(t, pontos);
  ctx.lineTo(p.x, p.y);
}

Algoritmo de De Casteljau

De Casteljau é uma forma geométrica de calcular B(t) — interpola linearmente pares adjacentes de pontos de forma recursiva até restar um único ponto:

Nível 0 (original): P₀ P₁ P₂ P₃ ↓ lerp(t) entre pares adjacentes Nível 1: Q₀=(1-t)P₀+tP₁, Q₁=(1-t)P₁+tP₂, Q₂=(1-t)P₂+tP₃ ↓ lerp(t) entre pares adjacentes Nível 2: R₀=(1-t)Q₀+tQ₁, R₁=(1-t)Q₁+tQ₂ ↓ lerp(t) Nível 3: B(t) = (1-t)R₀ + t·R₁ ← ponto na curva
javascript curvas/bezier.html 
function deCasteljau(t, pts) {
  let atual = pts.slice();

  while (atual.length > 1) {
    const proximo = [];
    for (let i = 0; i < atual.length - 1; i++) {
      // Interpolação linear entre pontos adjacentes
      proximo.push({
        x: (1 - t) * atual[i].x + t * atual[i + 1].x,
        y: (1 - t) * atual[i].y + t * atual[i + 1].y,
      });
    }
    atual = proximo;   // desce um nível
  }

  return atual[0];   // ponto na curva para este t
}

Parte 3 — Transformações 2D Three.js

Tecnologia: Three.js com OrthographicCamera (câmera sem perspectiva, correta para CG 2D).

Por que coordenadas homogêneas?

Com matrizes 2×2 conseguimos rotação e escala, mas não translação. A solução é adicionar uma coordenada extra w=1:

Ponto 2D normal: [x, y] Ponto homogêneo: [x, y, 1] (w = 1 para pontos, w = 0 para vetores) Transformação: P' = M · P Com w=1, a translação vira multiplicação de matriz — elegante e unificado.

3.1 — Translação

Arquivo: transformacoes2d/translacao.html

Matriz T(tx, ty): | 1 0 tx | | x | | x + tx | | 0 1 ty | × | y | = | y + ty | | 0 0 1 | | 1 | | 1 |
javascriptThree.js
// Three.js representa a translação como posição do objeto:
mesh.position.x += tx;
mesh.position.y += ty;

// Equivalente — método nativo:
mesh.translateX(tx);
mesh.translateY(ty);

Controles: setas ← → ↑ ↓ ou botões. Campo "Passo" define a magnitude.

3.2 — Rotação

Arquivo: transformacoes2d/rotacao.html

Rotação ao redor da ORIGEM R(θ): | cos θ −sin θ 0 | | x | | x·cos θ − y·sin θ | | sin θ cos θ 0 | × | y | = | x·sin θ + y·cos θ | | 0 0 1 | | 1 | | 1 | Propriedades: R(θ) é ortogonal: R⁻¹(θ) = Rᵀ(θ) = R(−θ) Preserva distâncias (isometria) Ângulo positivo = sentido anti-horário

Rotação ao redor de ponto arbitrário (px, py)

M = T(px, py) · R(θ) · T(−px, −py) 1. Translada (px,py) para a origem 2. Rotaciona 3. Translada de volta
javascripttransformacoes2d/rotacao.html
function girar(sentido) {
  const delta = sentido * passo_em_radianos;
  const px = pivX, py = pivY;

  // Posição atual em relação ao pivô
  const dx = mesh.position.x - px;
  const dy = mesh.position.y - py;

  const cosA = Math.cos(delta);
  const sinA = Math.sin(delta);

  // Aplica R(θ) à posição relativa ao pivô: T·R·T⁻¹
  mesh.position.x = px + dx * cosA - dy * sinA;
  mesh.position.y = py + dx * sinA + dy * cosA;

  // Rotaciona o próprio objeto
  mesh.rotation.z += delta;
}

Controles: teclas Q (−θ) e E (+θ). Campos "Piv. X/Y" alteram o pivô.

3.3 — Escala

Arquivo: transformacoes2d/escala.html

Matriz S(sx, sy): | sx 0 0 | | x | | sx · x | | 0 sy 0 | × | y | = | sy · y | | 0 0 1 | | 1 | | 1 | Casos especiais: sx = sy → escala uniforme (preserva forma) sx ≠ sy → escala não-uniforme (distorce) 0 < sx < 1 → reduz horizontalmente sx > 1 → amplia horizontalmente sx = −1 → reflexão em relação ao eixo Y sy = −1 → reflexão em relação ao eixo X sx = sy = −1 → rotação de 180° (equivalente)

// Escala ao redor de ponto arbitrário: T·S·T⁻¹

3.4 — Composição de Transformações

Arquivo: transformacoes2d/composicao.html

Atenção: matrizes de transformação não comutam em geral. T·R ≠ R·T. A ordem de composição define o resultado.
Aplicar S, depois R, depois T: M = T · R · S A multiplicação lê-se da direita para esquerda: 1. S é aplicado primeiro (escala no espaço local) 2. R é aplicado depois (rotação) 3. T é aplicado por último (translação) Transformação mundo de um nó filho: M_mundo = M_pai · M_avô · ... · M_local_filho

Parte 4 — Viewing 2D Janela → Viewport

Arquivo: viewing/janela_viewport.html

O processo de exibição 2D mapeia coordenadas do espaço do mundo (janela) para pixels da tela (viewport):

Mundo (World Space) Tela (Screen Space) ┌──────────────────┐ ┌──────────────────┐ │ │ │ │ │ Janela │ ──mapeamento──→ │ Viewport │ │ (wx, wy) │ │ (vx, vy) px │ │ coords lógicas │ │ pixels de tela │ └──────────────────┘ └──────────────────┘ wx ∈ [wxMin, wxMax] vx ∈ [vxMin, vxMax] wy ∈ [wyMin, wyMax] vy ∈ [vyMin, vyMax]

Fórmula de Mapeamento

Fatores de escala: sx = (vxMax − vxMin) / (wxMax − wxMin) sy = (vyMax − vyMin) / (wyMax − wyMin) Mapeamento de ponto (xw, yw) → (xv, yv): xv = vxMin + (xw − wxMin) · sx yv = vyMin + (yw − wyMin) · sy

Esta é uma interpolação linear (lerp) — normaliza o ponto dentro da janela e re-escala para o viewport.

Inversão do eixo Y

Em coordenadas matemáticas Y cresce para cima; em canvas Y cresce para baixo. A correção é usar (wyMax − yw) em vez de (yw − wyMin):

yv = vyMin + (wyMax − yw) · sy ← inversão do eixo Y

Implementação — janela_viewport.html

javascriptviewing/janela_viewport.html
function janelaParaViewport(xw, yw, janela, viewport) {
  // Fatores de escala: razão viewport / janela
  const sx = (viewport.xMax - viewport.xMin)
           / (janela.xMax   - janela.xMin);
  const sy = (viewport.yMax - viewport.yMin)
           / (janela.yMax   - janela.yMin);

  const xv = viewport.xMin + (xw - janela.xMin) * sx;
  // Inverte Y — canvas cresce para baixo, mundo cresce para cima
  const yv = viewport.yMin + (janela.yMax - yw) * sy;

  return { x: xv, y: yv };
}

Distorção por proporções diferentes

Janela: largura 20 × altura 10 → razão 2:1 Viewport: 400px × 400px → razão 1:1 sx = 400/20 = 20 (1 unidade = 20 pixels em X) sy = 400/10 = 40 (1 unidade = 40 pixels em Y) sx ≠ sy → distorção! Para preservar: usar sx = sy = min(sx_calc, sy_calc)

Em janela_viewport.html, a opção "Manter proporção" calcula automaticamente vy_max para evitar distorção.

Referência Rápida — Matrizes 3×3

Todas as transformações afins 2D em coordenadas homogêneas:

Translação T(tx, ty)
| 1 0 tx | | 0 1 ty | | 0 0 1 |
Escala S(sx, sy)
| sx 0 0 | | 0 sy 0 | | 0 0 1 |
Rotação R(θ)
| cosθ −sinθ 0 | | sinθ cosθ 0 | | 0 0 1 |
Reflexão — eixo X
| 1 0 0 | | 0 −1 0 | | 0 0 1 |
Reflexão — eixo Y
| −1 0 0 | | 0 1 0 | | 0 0 1 |
Cisalhamento Sh(shx, shy)
| 1 shx 0 | | shy 1 0 | | 0 0 1 |
Identidade I
| 1 0 0 | | 0 1 0 | | 0 0 1 |
Composição S→R→T
M = T · R · S (direita para esquerda: S primeiro, depois R, T por último)

Glossário — Conceitos-Chave

ConceitoDefinição
RasterizaçãoConverter geometria contínua em pixels discretos numa grade
Eixo dominanteO eixo com maior variação; define o número de passos nos algoritmos de linha
Variável de decisão PInteiro acumulado em Bresenham que decide qual pixel é mais próximo da curva ideal
Coord. homogêneasAdição da coordenada w para representar translação como multiplicação de matriz
Composição de matrizesProduto de matrizes de transformação; a ordem de aplicação importa
Fecho convexoMenor polígono convexo contendo todos os pontos de controle; a curva de Bézier está contida nele
Parâmetro tValor em [0,1] que percorre a curva de Bézier do ponto inicial (t=0) ao final (t=1)
Polinômio de BernsteinPeso C(n,i)·tⁱ·(1−t)^(n−i) que pondera cada ponto de controle
Janela (Window)Região do espaço-mundo que está sendo visualizada (coordenadas lógicas)
ViewportRegião de pixels na tela onde o conteúdo da janela é projetado
OrthographicCameraCâmera sem perspectiva — objetos não ficam menores com a distância; correta para 2D
THREE.GroupNó vazio no Three.js que propaga suas transformações para todos os objetos filhos
